# Competitive positioning

memory.cpp is a local-first AI memory and context control plane.

It overlaps with persistent memory tools, context APIs, stateful agent systems, provider SDK workflows, and coding-agent context files. The difference is the product lane:

- local by default
- no telemetry
- no accounts
- inspectable memory vault
- context compiler instead of generic top-k RAG
- token firewall and KV pressure estimates
- provider pack generation for coding tools
- memory tests and trust reports

memory.cpp complements AI providers and runtimes. It does not replace model serving engines, hosted memory APIs, or provider-side cache systems.
